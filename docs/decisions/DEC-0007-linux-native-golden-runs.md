# DEC-0007 — Финализация Linux golden runs на native Linux (x86_64)

## Контекст
- Для дифф‑harness и последующих шагов (, ) нужен достоверный Linux эталон.
- Предварительные наборы, собранные не в native окружении, могут отличаться путями, локалью, зависимостями и поведением terminal/UI.
- Риск RISK-0023 фиксировал необходимость пересъёма.

## Решение
Снять и зафиксировать Linux golden runs в native Linux окружении (x86_64) и использовать этот архив как источник истины для Linux.

## Альтернативы
A) Использовать предварительные не‑native эталоны.
- Минусы: повышенная вероятность ложных расхождений.

B) Полностью отказаться от Linux эталонов и полагаться только на unit‑тесты.
- Минусы: unit‑тесты не покрывают весь CLI‑контракт и интеграционные сценарии.

## Обоснование выбора
- Native прогоны обеспечивают корректную платформенную базу для 1:1 сравнения.

## Последствия
- Архив Linux эталонов становится обязательным входом для (diff harness) и для регрессионных сравнений.

## Критерии проверки/пересмотра
- Наличие артефакта Projects `rust_golden_runs_linux_x86_64.zip` со структурой `env.json`, `build.json`, `manifest.json`, `RUN-*`.
