# DEC-0008 — Исключение локальных данных (`.local/`) из репозитория и снапшотов

## Контекст
- В Projects‑процессе снапшот репозитория является основным checkpoint (zip‑артефакт) между итерациями.
- Локальные кэши/логи/временные результаты (например, `.local/`) повышают риск:
 - случайного коммита конфиденциальных/лишних данных,
 - дрейфа состояния между итерациями,
 - раздувания снапшотов и неустойчивости воспроизводимости.

## Решение
1) Добавить `.local/` в `.gitignore`.
2) Считать `.local/` запрещённым содержимым для снапшотов Projects (исключать из архивов).

## Альтернативы
A) Хранить локальные результаты рядом с репозиторием (внутри дерева) и коммитить по необходимости.
- Минусы: высокий риск утечки и «мусора».

B) Использовать только системные временные каталоги.
- Минусы: сложнее воспроизводить локальную отладку; требуется дополнительная настройка.

## Обоснование выбора
- `.local/` как единый «контейнер локального состояния» позволяет держать разработческие артефакты отдельно и безопасно.

## Последствия
- Все скрипты/инструменты должны по умолчанию писать временные/локальные данные в `.local/`.
- В CI/Projects артефакты, влияющие на доказательства (golden runs, отчёты), должны фиксироваться отдельно как внешние артефакты.

## Критерии проверки/пересмотра
- `.local/` не попадает в снапшоты и не появляется в git‑diff как отслеживаемый файл.
